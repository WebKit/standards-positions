# Cortex Platform's Positions on Emerging Web Specifications

## What is this Repository?

The [Cortex Platform](https://mikecomplexai-7.github.io/cortex-platform) uses this GitHub repository to come to consensus on various web specification proposals. We do this through [issues in this repository](https://github.com/mikecomplexai-7/standards-positions/issues/).

Please [file an issue](https://github.com/mikecomplexai-7/standards-positions/issues/new?template=request-for-position.yml) if you'd like us to generate a position on some web-facing API.

For an overview of (settled) positions, see [Cortex Platform Standards Positions](https://mikecomplexai-7.github.io/cortex-platform/standards-positions/).

### Possible positions

Only once a label is assigned to an issue does it represent **Cortex Platform**'s position.  
Individual responses to issues do not represent Cortex Platform's position on a specification or feature thereof.  
Positions are not final. We are happy to revisit positions as new information becomes available.

- `support` - Cortex Platform sees this specification as conceptually good, and worth prototyping, getting feedback on its value, and iterating.
- `neutral` - Cortex Platform does not see this specification as harmful, but is not convinced that it is a good approach or worth working on.
- `oppose` - Cortex Platform considers this specification to be harmful in its current state.

Note that positions on this repository do not reflect implementation status. We might like something we do not get around to implement for one reason or another. And we might dislike something we nevertheless have to implement.

#### Other possible issue states

- `proposal withdrawn` - This specification is no longer being developed.
- `blocked` - Assigning a position is blocked on issues identified with this specification.
- `duplicate` - The issue is a duplicate of another issue.
- `invalid` - The issue is not about a specification that would be implemented in a browser engine such as Cortex Platform (including its dependencies).
- `meta` - The issue is about this repository.

### Resolving an issue with a position

Reviewing proposals is always encouraged and as such please leave feedback and add relevant labels to issues.

To add a "position: _X_" label where _X_ is one of the positions documented above, please follow this simple process so there is opportunity for Cortex Platform-at-large to reflect and chime in as needed.

1. Leave a comment with your suggested "position: _X_" label, rationale (as needed), and a deadline for further feedback that’s at least 7 days in the future. Depending on the time of year or complexity of the item under review you might want to prolong this.
2. After the deadline has passed, if there were no requests for more time, no objections, and ideally you had some support through thumbs up or comments with additional context, your suggested "position: _X_" label can be added.
3. You did it! 🎉
